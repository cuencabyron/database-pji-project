# TABLA CUSTOMER 
| Campo                         | Tipo                                                                       | ¿Por qué así? 
| ----------------------------- | -------------------------------------------------------------------------- | -----------------------------                                                                                                                                            
| `customer_id`                  | `CHAR(36)`                                                                 | Almacena un **UUID v4** textual (36 chars con                                                                                                            guiones). Ventaja: unicidad global sin depender de autoincremento. Alternativa: `BINARY(16)` si quieres ahorrar espacio (guardando el UUID binario). |
| `nombre`                      | `VARCHAR(150)`                                                             | Nombre del cliente. `VARCHAR` ahorra espacio y 150 cubre nombres largos (personas o empresas).                                                                                                     |
| `email`                       | `VARCHAR(254)` `UNIQUE`                                                    | 254 es el **máximo estándar** definido por RFC para emails. `UNIQUE` evita duplicados.                                                                                                             |
| `telefono`                    | `VARCHAR(30)`                                                              | Los teléfonos no se tratan como número (pueden llevar `+`, paréntesis, espacios). Con 30 cubres internacional y extensiones.                                                                       |
| `direccion`                   | `TEXT`                                                                     | Puede crecer mucho (calle, colonia, referencias). `TEXT` evita límites artificiales.                                                                                                               |
| `activo`                      | `TINYINT(1)` (boolean)                                                     | En MySQL un booleano se representa como `TINYINT(1)`. 1 = activo, 0 = inactivo.                                                                                                                    |
| `creado_en`, `actualizado_en` | `DATETIME` con `DEFAULT CURRENT_TIMESTAMP` y `ON UPDATE CURRENT_TIMESTAMP` | Sellos de tiempo de alta y actualización. `DATETIME` mantiene zona horaria neutral en MySQL. `ON UPDATE` actualiza automáticamente.                                                                |


# TABLA SERVICES
| Campo         | Tipo                                  | ¿Por qué así?                                                                                                                                        |
| ------------- | ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id_servicio` | `CHAR(36)`                            | Igual razonamiento que en clientes (UUID).                                                                                                           |
| `nombre`      | `VARCHAR(150)`                        | Nombre comercial.                                                                                                                                    |
| `descripcion` | `TEXT`                                | Descripciones pueden ser largas.                                                                                                                     |
| `precio`      | `DECIMAL(12,2)` `CHECK (precio >= 0)` | Para **dinero**, nunca uses `FLOAT/DOUBLE` (errores de precisión). `DECIMAL(12,2)` soporta hasta 999,999,999,999.99. Con **check** evitas negativos. |
| `activo`      | `TINYINT(1)`                          | Permite desactivar sin borrar.                                                                                                                       |
| `creado_en`   | `DATETIME`                            | Sello de alta.                                                                                                                                       |

# TABLA SESSIONS
| Campo          | Tipo                             | ¿Por qué así?                                                                                                                       |
| -------------- | -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `id_sesion`    | `CHAR(36)`                       | UUID de sesión.                                                                                                                     |
| `cliente_id`   | `CHAR(36)` `FK`                  | Relación a `clientes`. Con `ON UPDATE CASCADE` y `ON DELETE CASCADE`: si cambias o borras el cliente, se reflejan las sesiones.     |
| `token_hash`   | `VARCHAR(255)` `UNIQUE`          | **Siempre** guarda **hash** del token (nunca el token en claro). 255 cubre SHA/Argon2/Bcrypt encodings. `UNIQUE` impide colisiones. |
| `refresh_hash` | `VARCHAR(255)`                   | Opcional, para refresh tokens, también **hash**.                                                                                    |
| `ip`           | `VARCHAR(45)`                    | 45 cubre IPv6 textual completa (ej. `2001:db8::1`).                                                                                 |
| `user_agent`   | `VARCHAR(300)`                   | UA puede ser largo. 300 alcanza para la mayoría, y evita `TEXT` si no lo necesitas.                                                 |
| `estado`       | `VARCHAR(20)` `DEFAULT 'activa'` | Estados como `activa`, `revocada`, `expirada`. Texto legible + flexible.                                                            |
| `inicio_en`    | `DATETIME`                       | Inicio real de la sesión.                                                                                                           |
| `fin_en`       | `DATETIME`                       | Cuando expiró/cerró. Nulo si sigue vigente.                                                                                         |
| `creado_en`    | `DATETIME`                       | Fecha de creación del registro.                                                                                                     |
| Índices        | `INDEX(cliente_id)`              | Para consultar rápido sesiones por cliente.                                                                                         |

# TABLA VERIFICATIONS
| Campo             | Tipo                                  | ¿Por qué así?                                                                                            |
| ----------------- | ------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| `id_verificacion` | `CHAR(36)`                            | UUID.                                                                                                    |
| `cliente_id`      | `CHAR(36)` `FK`                       | Relación con cliente (el dueño del proceso de verificación).                                             |
| `sesion_id`       | `CHAR(36)` `FK`, `SET NULL on delete` | Puede asociarse a una sesión concreta, pero si la sesión se borra, no pierdes el historial (queda null). |
| `tipo`            | `VARCHAR(30)`                         | Ej. `email`, `telefono`, `identidad`, `biometria`. Flexible ante nuevos métodos.                         |
| `estatus`         | `VARCHAR(20)` `DEFAULT 'pendiente'`   | `pendiente`, `aprobada`, `rechazada`, `expirada`.                                                        |
| `intentos`        | `INT` `DEFAULT 0`                     | Control anti-abuso y de flujo.                                                                           |
| `codigo_hash`     | `VARCHAR(255)`                        | Guarda **hash del OTP** o resultado, jamás el código plano.                                              |
| `expira_en`       | `DATETIME`                            | Fecha de caducidad del OTP/verificación.                                                                 |
| `verificado_en`   | `DATETIME`                            | Cuándo se concluyó/aprobó.                                                                               |
| `creado_en`       | `DATETIME`                            | Fecha de creación.                                                                                       |
| Índices           | `INDEX(cliente_id, estatus)`          | Para listar verificaciones activas/pendientes por cliente rápidamente.                                   |
